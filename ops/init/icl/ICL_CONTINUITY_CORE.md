# ICL Continuity Core

## Purpose
Define the minimal, authoritative continuity/rehydration surface for stateless AI operation in nukeCE.

## Core (pointer-only)
- `ops/init/icl/CONTEXT_PACK.md` — canonical read-in card.
- `ops/init/icl/AI_CONTEXT_SYNC.md` — SSOT "save game" for rehydration.
- `ops/init/icl/DAILY_CONSOLE.md` — operating rhythm (log, not canon).
- `ops/init/icl/RECOVERY.md` — copy/paste recovery prompts.
- `ops/init/icl/context_pack.json` — machine-readable context pointers.
- `ops/init/icl/principles.md` — core principles.
- `ops/init/icl/ASSISTANT_PROFILE.md` — assistant collaboration posture.
- `ops/init/icl/USER_PROFILE.md` — operator working preferences.
- `ops/init/icl/ACTIVE_CONTEXT.md` — pointer to the active core.

## Deprecation
Legacy onboarding bundles are deprecated in favor of the ICL Continuity Core.
Historical onboarding artifacts live in `ops/init/icl/deprecated/` and are not core.
