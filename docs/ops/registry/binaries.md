# Binary Registry

Authoritative registry for `ops/bin/*` entrypoints.

| ID | Name | File Path | Infra Importance | Notes |
| --- | --- | --- | --- | --- |
| OPS-BIN-01 | Context Session Stream | ops/bin/context | supporting | Spec: `docs/ops/specs/binaries/context.md`. Wrapper over `ops/lib/scripts/synthesize.sh`; defaults to OPS layer and injects OPEN session state. |
| OPS-BIN-02 | Dump | ops/bin/dump | critical | Spec: `docs/ops/specs/binaries/dump.md`. Format converter and compressor that consumes the traversal stream from `ops/lib/scripts/traverse.sh`, then emits receipt payloads and manifests. |
| OPS-BIN-03 | Help | ops/bin/help | supporting | Spec: `docs/ops/specs/binaries/help.md`. Surfaces operational docs and wayfinding pointers. |
| OPS-BIN-04 | LLMS Bundler | ops/bin/llms | important | Spec: `docs/ops/specs/binaries/llms.md`. Wrapper over `ops/lib/scripts/synthesize.sh`; writes only `llms.txt`, `llms-core.txt`, and `llms-full.txt`. |
| OPS-BIN-05 | MAP Generator | ops/bin/map | supporting | Spec: `docs/ops/specs/binaries/map.md`. Refreshes the auto-generated MAP index block. |
| OPS-BIN-06 | Open | ops/bin/open | critical | Spec: `docs/ops/specs/binaries/open.md`. Produces session-state OPEN artifacts with freshness details and canonical wrapper markers `===== STELA OPEN PROMPT =====` / `===== END STELA OPEN PROMPT =====`; OPEN is pointer-first for dirty porcelain detail, which is retained in OPEN-PORCELAIN artifacts. |
| OPS-BIN-07 | Bundle | ops/bin/bundle | critical | Spec: `docs/ops/specs/binaries/bundle.md`. Canonical front door and deterministic transport primitive for `auto|planning|architect|audit|project|conform|foreman` profiles, with legacy `hygiene` accepted as a `sunset` alias to `conform`. Alias routing and front-door policy are manifest-driven from `ops/lib/manifests/BUNDLE.md`. Architect transport is direct plan intake from `storage/handoff/PLAN.md` with request metadata (`request.plan_source`, `request.packet_id`, `request.dp_draft_path`, `request.closing_sidecar`). Supports optional ATS triplet validation (`--agent-id`, `--skill-id`, `--task-id`) using `ops/lib/manifests/ASSEMBLY.md`, emits assembly metadata in the bundle manifest, and emits deterministic runtime assembly pointer artifacts when ATS is applied. Emits bundle `.txt`, `.manifest.json`, and `.tar` sidecar under `storage/handoff/`. |
| OPS-BIN-08 | Prune | ops/bin/prune | important | Spec: `docs/ops/specs/binaries/prune.md`. Cleans local artifacts. |
| OPS-BIN-09 | Manifest Compiler | ops/bin/compile | supporting | Spec: `docs/ops/specs/binaries/compile.md`. Compiles manifest templates from `ops/src/manifests/` into deterministic explicit outputs in `ops/lib/manifests/`. |
| OPS-BIN-10 | Template Compatibility Router | ops/bin/template | supporting | Spec: `docs/ops/specs/binaries/template.md`. Deprecated renderer shim retained for backward compatibility; uses `exec` dispatch to `ops/bin/factory` (`agent`, `task`, `skill`) and `ops/bin/manifest` (`dp`, `results`, `task-surface`, `spec`, stance render keys). Compatibility stance aliases such as `stance-analyst` remain shim-valid so existing render pipelines continue to resolve. |
| OPS-BIN-11 | Certifier | ops/bin/certify | critical | Spec: `docs/ops/specs/binaries/certify.md`. Runs integrity and receipt verification gates, then renders and validates DP RESULTS artifacts; intake fallback is explicit-only via `--allow-intake-fallback`; supports non-interactive narrative ingestion via `--narrative-file=PATH` and sources `ops/lib/scripts/editor.sh` for narrative capture/validation. |
| OPS-BIN-12 | Factory Backend | ops/bin/factory | important | Spec: `docs/ops/specs/binaries/factory.md`. Factory Semantics backend for template types `agent`, `task`, and `skill`. Depends on `ops/lib/scripts/common.sh` and is dispatched by `ops/bin/template` rather than called directly by existing call sites. |
| OPS-BIN-13 | Manifest Backend | ops/bin/manifest | supporting | Spec: `docs/ops/specs/binaries/manifest.md`. Compiler Semantics backend for template types `dp`, `results`, `task-surface`, and `spec`. Depends on `ops/lib/scripts/common.sh` and is dispatched by `ops/bin/template` rather than called directly by existing call sites. |
| OPS-BIN-14 | Scaffold | ops/bin/scaffold | supporting | Spec: `docs/ops/specs/binaries/scaffold.md`. Project directory provisioning entrypoint. Creates `projects/<name>/` structure per `ops/lib/project/SCAFFOLD.md`, validates project-name format, and enforces uniqueness. |
| OPS-BIN-15 | Meta | ops/bin/meta | supporting | Spec: `docs/ops/specs/binaries/meta.md`. Project-only compatibility shim. Enforces project front-door contract and front-door policy keys, then invokes `ops/bin/bundle --profile=project --project=<name> --out=<resolved-value>` while preserving the existing meta CLI and optional explicit output-path passthrough. |
| OPS-BIN-16 | DP Draft Generator | ops/bin/draft | supporting | Spec: `docs/ops/specs/binaries/draft.md`. Generates immutable DP structure in `TASK.md` from `ops/src/surfaces/dp.md.tpl`; adds scaffold assist flags for plan/draft-slots emit-edit-validate flows without mutating TASK outside generation mode. |
| OPS-BIN-17 | Addendum Generator | ops/bin/addendum | supporting | Spec: `docs/ops/specs/binaries/addendum.md`. Renders fully-formed addendum artifacts from `ops/src/stances/addendum.md.tpl` using a slots file; assigns next addendum letter automatically; validates required slot population before writing to `storage/dp/intake/`. Issued by the Integrator/foreman authority role; Contractor does not invoke this binary. |
| OPS-BIN-18 | CbC Escalation Report | ops/bin/cbc | supporting | Spec: `docs/ops/specs/binaries/cbc.md`. Reads the decision registry and reports B-scored entries older than the staleness threshold (default: six months, two quarters). Supports `--registry`, `--threshold-months`, and `--as-of` flags. Informational only; exits 0 always. |
| OPS-BIN-19 | Decision Leaf Generator | ops/bin/decision | important | Spec: `docs/ops/specs/binaries/decision.md`. Writes RoR decision leaves and updates `RoR.md` so review outcomes are durable canon instead of narrative-only notes. |
