# Calibrated Canonical Density (CCD)

## 0. Why This Exists
Calibrated Canonical Density defines a simple idea: important writing must carry enough meaning per sentence to justify its place in the canon. The goal is reliable transfer of intent between people and machines with minimal drift. When density is too low, readers spend effort on filler and still miss decisions. When density is too high, readers see compressed language that hides assumptions. CCD sets a shared contract so every author can aim for the same operating zone.

## 1. The Problem Before CCD
Before a density contract existed, review outcomes were inconsistent. One reviewer might call a file clear while another might call the same file verbose or abstract. Both views could be reasonable because no measurable target existed. Style disputes consumed review time without improving operational truth. Governance surfaces drifted into mixed audiences where experts could decode intent but newer operators could not. CCD closes this gap by naming density as an explicit property that can be declared, checked, and discussed.

## 2. The Contract Model
CCD treats density as a declaration, not a hidden preference. A surface states an intended target and a tolerance band. The declaration does not replace human judgment. It scopes judgment so discussion starts from a known contract instead of personal taste. This model makes disagreements productive. If text falls outside its declared band, the conversation is concrete: either the writing must change, or the declared band must be corrected with a reason. The system now asks a factual question rather than a subjective one.

## 3. Tier Roles and Boundaries
CCD follows tiered responsibility. Tier 1 defines hard governance constraints. Tier 2 defines executable mechanics and checks. Tier 3 explains rationale and meaning for humans. This file is Tier 3 by design. It explains why the contract exists and how to reason about it. It does not define execution order, command routing, or implementation details. Keeping tiers separate protects clarity and reduces accidental cross-layer edits.

## 4. Why Density Improves Reliability
Context windows are finite. Every low-signal sentence competes with critical state. A density contract improves reliability because it aligns authors around information value per line. Readers can predict what kind of detail belongs in each surface and what does not. This supports the Policy of Truth mandates for linguistic precision and relatability. Precision is protected because claims must be direct and testable. Relatability is protected because a readable density band discourages obscure compression that only specialists can parse.

## 5. Domain Includes Consumer
CCD resolves a common conflict: should writing optimize for domain experts or for general operators. The answer is domain includes consumer. Canon text must preserve domain correctness while staying legible to the intended consumer of that surface. Density contracts make that balance explicit. A very dense expert memo might be valid in a narrow context, but it is not valid by default for shared governance surfaces. Conversely, overly simplified text that drops crucial constraints is also invalid. CCD keeps both failure modes visible.

## 6. Operating Outcome
A successful CCD culture produces predictable documents. New operators can explain why a rule exists, not only repeat the rule. Reviewers spend less time debating tone and more time validating truth claims. Tooling can flag likely density drift early while still leaving room for informed exceptions. Over time, canon surfaces become easier to load and easier to maintain. CCD makes clarity an enforceable property of the system rather than an accidental result of individual writing style.
