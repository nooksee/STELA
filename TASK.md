## 3. Current Dispatch Packet (DP)

Section-by-section authoring protocol:
- Write Section 3.1. Run the gate. Stop on mismatch. Freeze 3.1.
- Write Section 3.2. Confirm canon load. Freeze 3.2.
- Write Section 3.3. Lock scope and allowlist. Freeze 3.3.
- Write Section 3.4 in order (3.4.1 through 3.4.5). Freeze each subsection when complete.
- Do not revise earlier sections to justify later work. If scope changes, end the thread and draft a new DP.

Freeze Rule:
- If a completed DP subsection changes later, append an AMENDMENT block in RESULTS; otherwise DP is FAIL.

AMENDMENT: minimum fields
- Reopened section:
- Reason:
- Impact:
- Allowlist change (none or exact paths):

Input discipline (hard rule):
- Worker receives only the DP text. Worker does not require OPEN or DUMP attachments.
- DP writer must not attach or cite disposable artifacts (rough notes, chat logs, scratch drafts, throwaway excerpts).
- DP writer must not include pasted bundles (OPEN, DUMP payloads, manifests) inside the DP body.
- Canon pointers only. Evidence belongs in storage artifacts and RESULTS.
