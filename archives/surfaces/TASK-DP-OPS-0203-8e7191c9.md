## 3.3 Scope and Safety
Objective:
- Make audit reruns unambiguous by giving each delivered resubmission a distinct artifact identity and explicit supersession metadata.
- Keep the delivered audit packet self-consistent on packet source, RESULTS, CLOSING, TASK leaf, manifest identity, and package membership.
- Remove the need for prose-only stale-packet correction when the real problem is attachment identity reuse.

In scope:
- UPDATE audit bundle naming so successive delivered reruns use distinct artifact identities instead of reusing the same `AUDIT-...` basename.
- UPDATE audit manifest metadata so reruns record lineage fields such as submission kind, resubmission index, superseded artifact identity, and refresh reason.
- UPDATE audit dump naming so dump payload and dump manifest identities track the same rerun lineage as the bundle artifact.
- UPDATE bundle transport logic so audit reruns are first-class delivery actions rather than ad hoc filename edits or correction-note workarounds.
- UPDATE smoke-quarantine routing so ephemeral smoke outputs live under var/tmp/_smoke rather than storage/_smoke, matching PoT payload-versus-resume canon.
- UPDATE bundle smoke to prove initial audit submission versus audit resubmission identity and fail on stale-name reuse.
- UPDATE docs and manual text so the audit rerun contract is explicit.
- UPDATE `storage/dp/active/allowlist.txt` only for tracked files touched by this packet.

Out of scope:
- Persistence manifest rename.
- Simple latest-wins shipping surface renames.
- Analyst, architect, or conform resubmission lineage.
- Auditor prompt changes beyond what the delivered transport itself makes unnecessary.

Safety and invariants:
- The first delivered audit packet for a packet keeps the normal `AUDIT-...` identity.
- Later delivered audit reruns for the same packet must not reuse the original artifact basename.
- Audit rerun identity must be visible in both filenames and manifest metadata.
- Dump payload, dump manifest, bundle text, bundle manifest, and package membership must agree on the same rerun identity.
- Audit delivery must stay fail-closed on ambiguous packet source resolution.
- The transport fix must reduce reliance on prose correction notes, not add a second competing truth surface.
- Payload-vs-resume canon must hold: operator delivery artifacts stay in storage/, while smoke scratch belongs in var/tmp/.
