## Closing Block Style Guide

The following rules govern authorship of the six Mandatory Closing Block entries in
`storage/handoff/CLOSING-*.md` files. These rules apply to all DPs authored after
DP-OPS-0080. Violations are enforced by `tools/lint/style.sh`.

1. Each of the six Mandatory Closing Block entries must use a distinct sentence structure.
   No two entries may open with the same lead verb or the same grammatical form.
2. The Primary Commit Header must be an imperative clause and must not echo the Pull Request
   Title verbatim. At minimum, the verb or the object must differ.
3. The Pull Request Description must contain at least one sentence explaining why the change
   is made, not only what changed.
4. The Final Squash Stub must differ from the Primary Commit Header in both verb and subject.
5. The Extended Technical Manifest must enumerate at least the primary changed files and
   the verification commands run; it must not be a restatement of the PR Description.
6. The Review Conversation Starter must pose a genuine open question to the reviewer - not
   a summary or a rhetorical affirmation. The question must be answerable by someone who has
   read only the diff.
